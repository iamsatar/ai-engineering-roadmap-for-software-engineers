# Contributing

Contributions are welcome if they make the roadmap more useful, more accurate, or more practical for experienced software engineers.

## Contribution Principles

- Prefer project-based learning over passive reading.
- Keep the roadmap applied and role-relevant.
- Avoid beginner programming material.
- Avoid deep learning from scratch unless clearly optional.
- Add resources sparingly.
- Include evaluation, observability, and safety whenever possible.

## Good Contributions

- Better mini-project specs
- Stronger capstone requirements
- Improved checklists
- Clearer architecture templates
- Realistic eval examples
- Security and safety scenarios
- Role-specific interview preparation

## Pull Request Checklist

- The contribution fits the applied AI engineering scope.
- New resources are not random link dumps.
- Project additions include deliverables and quality criteria.
- Templates remain easy to reuse.
- Language is clear and concise.

## Style

- Write for senior software engineers.
- Be direct and practical.
- Use concrete deliverables.
- Prefer examples over vague advice.
