# Repo Quality Checklist

- [ ] README renders cleanly.
- [ ] All tables render correctly.
- [ ] All track links work.
- [ ] Every track has `doc.md`.
- [ ] Every track has `mini-project.md`.
- [ ] Every mini-project has acceptance criteria.
- [ ] RESOURCES.md has real links.
- [ ] PROJECTS.md has capstone requirements.
- [ ] JOB_READINESS.md has practical application criteria.
- [ ] Examples folder exists.
- [ ] Case-studies folder exists.
- [ ] No fake metrics are included.
- [ ] No secrets or API keys are included.
- [ ] Capstone count is consistent.
- [ ] README has GitHub metadata suggestions.
- [ ] GitHub repo settings have description and topics applied.
