# Project Completion Checklist

- [ ] The project runs locally from documented commands.
- [ ] Environment variables are documented.
- [ ] The README explains the problem and workflow.
- [ ] Model calls are logged or traceable.
- [ ] Costs and latency are visible.
- [ ] There are success and failure test cases.
- [ ] The project handles invalid model output.
- [ ] The project handles API and tool failures.
- [ ] Security risks are documented.
- [ ] Known limitations are documented.
- [ ] A build-log entry exists.
