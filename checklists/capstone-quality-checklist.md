# Capstone Quality Checklist

- [ ] The app solves a realistic workflow.
- [ ] The demo is easy to understand in under five minutes.
- [ ] The architecture doc is complete.
- [ ] The eval report includes real examples and failure analysis.
- [ ] The system has observability for model calls, tool calls, cost, and latency.
- [ ] The security report covers prompt injection and data exposure.
- [ ] The system has at least one human approval gate where appropriate.
- [ ] The README includes limitations and future work.
- [ ] The project has clear resume bullets.
- [ ] You can explain the design in a system design interview.
