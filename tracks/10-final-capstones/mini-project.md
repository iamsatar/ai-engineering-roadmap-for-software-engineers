# Mini Project — Final Capstone Portfolio

## Goal

Package at least one production-grade capstone into a portfolio-ready case study.

## What It Teaches

Project scoping, architecture documentation, eval reporting, observability, security notes, demo
writing, and interview storytelling.

## Suggested Stack

Use the stack appropriate for the chosen capstone. TypeScript, React, Node.js, a model provider SDK,
a database, tracing, and deployment tooling are common choices.

## Requirements

- Choose one recommended capstone.
- Build a working application.
- Add evals and observability.
- Document cost and latency.
- Document security risks and mitigations.
- Provide demo video or screenshots.
- Write resume bullet points.

## Acceptance Criteria

- [ ] The app solves a realistic workflow.
- [ ] Another engineer can run or inspect it.
- [ ] README, architecture doc, eval report, and security notes exist.
- [ ] Cost and latency tradeoffs are explained.
- [ ] Demo video or screenshots exist.
- [ ] Resume bullets are ready.

## Stretch Goals

- Build a second capstone.
- Build all three recommended capstones.
- Add the optional LLM Gateway / Router capstone for platform roles.

## README Expectations

Include problem, demo, screenshots or video, architecture, data flow, how it works, setup,
environment variables, evaluation, observability, cost and latency, security, known limitations,
future improvements, and resume bullets.

## Estimated Time

4-8 weeks per substantial capstone.
